# SyncDesk

**Sistema Estruturado para Atendimento ao cliente, Triagem e Escalonamento de Demandas**

## Desafio

Desenvolvimento de uma plataforma mobile e web de atendimento e suporte técnico centralizado, que utiliza triagem automatizada e estruturação de dados para otimizar o fluxo de chamados. O sistema visa a organização de solicitações via chat, relacionando: perfil do cliente e histórico de interações, gerando uma classificação inteligente que direcione o atendimento ao grupo de suporte adequado (N1, N2 ou especializado). A solução foca em eliminar o retrabalho e a sobrecarga operacional, permitindo que a gestão acompanhe indicadores precisos e garanta uma jornada do usuário fluida, evitando gargalos no atendimento e assegurando a resolução eficiente das demandas tecnológicas.

## ⚡ Desenvolvimento Ágil

O projeto foi feito seguindo o método Ágil SCRUM, dividindo o trabalho em sprints de 21 dias, com reuniões diáras, revisões e retrospectivas ao final.

### 📋 Backlog do Produto

| Sprint | User Story                                                                                                                                    | Status    | Prioridade  |
| :----- | :-------------------------------------------------------------------------------------------------------------------------------------------- | :-------- | :---------- |
| **1**  | Como atendente, quero poder trocar mensagens com clientes em uma tela de chat, para conseguir compreender suas necessidades e auxiliar rapidamente. | CONCLUÍDA | ▲ Alta      |
| **1**  | Como solicitante, quero começar uma conversa rapidamente e ser guiado com algumas perguntas iniciais, para que meu problema seja identificado de forma ágil e descomplicada.                | CONCLUÍDA | ▲ Alta      |
| **1**  | Como atendente, quero gerenciar o ciclo de vida dos chamados (atualizar status, encaminhar, atribuir e encerrar), para acompanhar e controlar as solicitações dos clientes. | CONCLUÍDA | ▲ Alta      |
| **2**  | Como atendente, quero poder acessar uma lista de chamados passados e em andamento com seus respectivos status, para monitorar os atendimentos sob minha responsabilidade.                                                                     | Pendente | **=** Média |
| **2**  | Como gestor da aplicação, quero que o meu acesso possua ferramentas administrativas (como gestão de permissões, auditoria e relatórios), para ter controle sobre a operação do suporte.                                                        | Pendente | ▲ Alta      |
| **3**  | Como solicitante do atendimento, quero receber notificações quando houver atualizações nos meus chamados, para acompanhar o andamento dos atendimentos sem precisar acessar constantemente o sistema.                         | Pendente | **=** Média |
| **3**  | Como usuário do sistema, quero buscar conversas no histórico de atendimentos com base em critérios como nome do destinatário e data, para localizar atendimentos específicos de forma mais rápida.                 | Pendente | **=** Média |
| **3**  | Como solicitante do atendimento, quero poder deixar uma avaliação após o encerramento, para registrar meu nível de satisfação com o suporte prestado.                 | Pendente | **=** Média |
